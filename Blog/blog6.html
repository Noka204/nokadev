Promise chain & async/await: lỗi nuốt exception và race

JS2 Async Pitfall 25.10.2025 • Đọc nhanh ~ 12 phút

1) Mục tiêu & tóm tắt

Mục tiêu: Hiểu vì sao lỗi bị “nuốt” trong Promise chain/async, cách bắt lỗi đúng chỗ, né race khi nhiều yêu cầu cùng lúc, và biết áp timeout/hủy request.

Tóm tắt: Luôn return trong .then, dùng try/catch đúng phạm vi cho await, kiểm soát race bằng “token/đếm phiên” hoặc AbortController, và chọn Promise.all/allSettled/any đúng mục tiêu.

2) Ý tưởng cốt lõi: Promise, microtask, async/await

  • Promise là “lời hứa” về một giá trị tương lai. Trạng thái: pending → fulfilled/rejected.
  • Microtask (job queue) chạy sau tick hiện tại: then/catch/finally là microtask.
  • async/await chỉ là “đường thẳng hoá” Promise: await chờ Promise settle; async luôn trả về Promise.

3) “Nuốt exception” trong chain & async

3.1 Quên return trong .then → chain đứt 

// Sai: không return promise bên trong .then → lỗi không nổi lên .catch
fetch('/api/a')
  .then(res => res.json())
  .then(data => {
    // gọi tiếp nhưng KHÔNG return
    fetch('/api/b?id=' + data.id)
      .then(r => r.json())
  })
  .catch(err => console.error('bắt không được', err))

Sửa: luôn return Promise để nối chuỗi. 

fetch('/api/a')
  .then(r => r.json())
  .then(a => fetch('/api/b?id=' + a.id))
  .then(r => r.json())
  .then(b => console.log('ok', b))
  .catch(err => console.error('lỗi', err))

3.2 async/await: try/catch không đủ rộng 

// Sai: chỉ catch dòng đầu, lỗi ở dòng 2 bị rơi
async function load() {
  try {
    const a = await fetch('/a')
  } catch (e) {}
  const b = await fetch('/b') // lỗi ở đây không ai bắt
}

Sửa: bao phạm vi đủ dùng hoặc bắt cục bộ từng bước có chủ đích. 

async function load() {
  try {
    const a = await fetch('/a')
    const b = await fetch('/b')
    return { a, b }
  } catch (e) {
    // log / báo UI
    throw e // cho caller xử lý tiếp
  }
}

3.3 Nuốt lỗi bằng .catch nhưng không rethrow

// Sai: catch xong im lặng → phía gọi tưởng là thành công
async function getData() {
  return fetch('/api').catch(e => {
    console.error('log', e)
    // thiếu: throw e
  })
}

Sửa: hoặc trả giá trị fallback rõ ràng, hoặc throw lại để tầng trên biết. 

async function getData() {
  try {
    const r = await fetch('/api')
    return r
  } catch (e) {
    console.error('log', e)
    throw e
  }
}

4) Race condition: nhận diện & hoá giải

Race xảy ra khi nhiều request “đua” nhau, kết quả đến trễ ghi đè kết quả mới. 

// người dùng gõ tìm kiếm: q1 → q2 nhanh
let lastReq = 0
async function search(q) {
  const req = ++lastReq
  const r = await fetch('/search?q=' + encodeURIComponent(q))
  const data = await r.json()
  if (req !== lastReq) return // kết quả đã lỗi thời
  render(data)
}

Giải pháp khác: AbortController để hủy request cũ. 

let ctrl
async function searchAbortable(q) {
  ctrl?.abort()
  ctrl = new AbortController()
  const r = await fetch('/search?q=' + encodeURIComponent(q), { signal: ctrl.signal })
  const data = await r.json()
  render(data)
}

5) Chạy song song vs tuần tự cho đúng

5.1 Tuần tự (có phụ thuộc)

async function loadSequential() {
  const a = await fetch('/a').then(r => r.json())
  const b = await fetch('/b?id=' + a.id).then(r => r.json())
  return { a, b }
}

5.2 Song song (độc lập)

async function loadParallel() {
  const pA = fetch('/a').then(r => r.json())
  const pB = fetch('/b').then(r => r.json())
  const [a, b] = await Promise.all([pA, pB])
  return { a, b }
}

6) Bắt lỗi theo hạt mịn

API Thành công khi Hành vi lỗi Dùng khi
Promise.all Tất cả fulfilled Chỉ cần 1 reject → reject ngay Các tác vụ phải cùng thành công
Promise.allSettled Luôn fulfilled Trả mảng {status, value|reason} Muốn báo cáo đủ: cái nào pass/fail
Promise.any Ít nhất 1 fulfilled Nếu tất cả reject → lỗi AggregateError Lấy kết quả nhanh nhất “thành công”
Promise.race Nhanh nhất settle Có thể fulfilled hoặc rejected Áp timeout hoặc chọn kết quả nhanh nhất

7) Timeout & AbortController

7.1 Tự cuốn timeout bằng race

const withTimeout = (p, ms, msg = 'Timeout') =>
  Promise.race([
    p,
    new Promise((_, rej) => setTimeout(() => rej(new Error(msg)), ms))
  ])

// dùng
withTimeout(fetch('/slow'), 3000)
  .then(r => r.json())
  .then(data => console.log(data))
  .catch(e => console.error(e.message))

7.2 Hủy thực sự bằng AbortController

const fetchWithTimeout = (url, ms) => {
  const ctrl = new AbortController()
  const id = setTimeout(() => ctrl.abort(), ms)
  return fetch(url, { signal: ctrl.signal })
    .finally(() => clearTimeout(id))
}

7.3 Bắt lỗi cho từng phần tử khi chạy nhiều promise

const safe = p => p.then(v => ({ ok: true, v })).catch(e => ({ ok: false, e }))

async function loadMany(urls) {
  const tasks = urls.map(u => safe(fetch(u).then(r => r.json())))
  const results = await Promise.all(tasks)
  const ok = results.filter(x => x.ok).map(x => x.v)
  const fail = results.filter(x => !x.ok).map(x => x.e)
  return { ok, fail }
}

8) Bài tập tự kiểm

Câu 1 — Return trong .then

fetch('/a')
  .then(r => r.json())
  .then(a => {
    // thêm đoạn gọi /b dựa trên a, và ĐẢM BẢO lỗi /b nổi lên .catch
  })
  .then(b => console.log(b))
  .catch(e => console.error('fail', e))

Đáp án gợi ý: return fetch('/b?id=' + a.id).then(r => r.json()).

Câu 2 — try/catch đủ rộng

async function run() {
  const a = await fetch('/a')
  const b = await fetch('/b')
}

Yêu cầu: Bắt lỗi cho cả 2 dòng mà vẫn rethrow cho caller.

9) Kết luận 3 câu

  1. Đừng nuốt lỗi: return trong .then, try/catch đúng phạm vi, và rethrow khi cần.
  2. Race condition rất âm thầm: chặn bằng “đếm phiên” hoặc AbortController.
  3. Chọn công cụ đúng mục tiêu: all/allSettled/any/race; áp timeout để UI không “treo”.
