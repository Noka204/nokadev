== vs === & truthy/falsy — 0, "", [] có “bằng nhau”?

JS1 Pitfall Quiz 25.10.2025 • Đọc nhanh ~ 9 phút

1) Mục tiêu & tóm tắt

Mục tiêu: Giải thích được cho một bạn chưa học JS: khác nhau giữa == và ===, vì sao 0, "", [] lúc “bằng”, lúc không; nắm truthy/falsy để không bị troll trong quiz NetAcad.

Tóm tắt: Dùng === mặc định. Dùng == chỉ khi cố ý: x == null để bắt null lẫn undefined.

2) Ý tưởng cốt lõi :

  • ===: so sánh nghiêm ngặt, không ép kiểu. Khác kiểu → false.
  • ==: so sánh lỏng, có ép kiểu theo quy tắc (string → number, object → primitive...). Dễ dính bẫy.
  • Truthy/Falsy: khi cần boolean (ví dụ trong if), JS ép sang true/false. Falsy: false, 0, -0, 0n, "", null, undefined, NaN. Còn lại là truthy (kể cả [], {}, "0"…)

3) Ép kiểu của ==: 0, "", []

Hãy nhớ đường đi thường gặp: [] (object) → ToPrimitive → "" (chuỗi) → rồi có thể → 0 (số). 

0 == ""      // true  ("" → 0)
0 === ""     // false (number vs string)

"" == []     // true  ([] → "" qua ToPrimitive)
"" === []    // false (string vs object)

0 == []      // true  ([] → "" → 0)
0 === []     // false

[] == false  // true  ([] → "" → 0 ; false → 0)
[] === false // false

[0] == 0     // true  ("0" → 0)
[1] == true  // true  ("1" → 1 ; true → 1)

[] == ![]    // true  (![] là false; suy ra [] == false → true)
Điểm mấu chốt: [] là truthy trong if, nhưng khi đem so sánh lỏng == nó bị ép thành "" → rồi thành 0. Hai ngữ cảnh khác nhau nên kết quả khác nhau.

4) Truthy/Falsy & khác biệt ngữ cảnh

if ("0")   console.log("chạy"); // chạy (truthy)
if ([])    console.log("chạy"); // chạy (truthy)
if ("")    console.log("không"); // không (falsy)
if (0)     console.log("không"); // không (falsy)

Khác biệt quan trọng: if (x) ép sang boolean; x == y ép kiểu để “bằng nhau”. Vì vậy [] có thể truthy trong if nhưng vẫn “bằng” 0 khi dùng ==.

5) Bảng tóm tắt tình huống hay gặp

So sánh Diễn giải Kết quả
0 == "" "" → 0 true
"" == [] [] → "" true
0 == [] [] → "" → 0 true
[] == false [] → "" → 0; false → 0 true
null == undefined quy tắc đặc biệt true
NaN == NaN NaN không bằng chính nó false
Object.is(NaN, NaN) so sánh “đồng nhất” true
0 === "0" khác kiểu false
"0" == false "0" → 0; false → 0 true

6) Thực chiến: viết code “an toàn”

6.1 Trước (dễ dính bẫy)

function toInt(x) {
  if (x == "") return 0;     // bẫy ép kiểu
  if (x == false) return 0;  // bẫy ép kiểu
  return parseInt(x);
}

6.2 Sau (rõ ràng, có chủ đích)

function toInt(x) {
  if (x === "") return 0;                    // kiểm tra tường minh
  if (typeof x === "boolean") return x ? 1 : 0;
  const n = Number(x);                       // ép kiểu chủ động
  return Number.isNaN(n) ? 0 : Math.trunc(n);
}

6.3 Quy tắc bỏ túi

  • Dùng ===/!== mặc định.
  • Ngoại lệ hợp lý: x == null (bắt cả null lẫn undefined).
  • Tránh == giữa số/chuỗi/boolean với mảng/đối tượng.
  • Cần boolean? Ép tường minh: Boolean(x) hoặc !!x.
  • NaN: dùng Number.isNaN(x); cần phân biệt 0 và -0? Object.is(x, -0).

7) Bài tập tự kiểm (đừng chạy máy trước)

Câu 1

0 == ""
0 === ""

Đáp án: true, false.

Câu 2

[] == ""
[] === ""

Đáp án: true, false.

Câu 3

[] == 0
[0] == 0
[1] == true

Đáp án: true, true, true.

Câu 4

[] == ![]

Đáp án: true (![] là false; [] == false → true).

Câu 5

Điền toán tử để biểu thức là true mà không viết dài dòng: 

const x = null;
const y = undefined;
// ...
x __ y

Đáp án: == (quy tắc đặc biệt cho null/undefined). Tương đương tường minh: (x === null && y === undefined).

Câu 6

Viết điều kiện kiểm tra “mảng không rỗng” chuẩn:

const a = [];
if ( ________ ) {
  // ...
}

Đáp án: Array.isArray(a) && a.length > 0.

8) Kết luận 3 câu

  1. === không ép kiểu, an toàn; == ép kiểu, nhiều cạm bẫy.
  2. "", 0, [] có thể “trượt” về 0 trong ==; nhưng [] vẫn truthy trong if.
  3. Mặc định dùng ===; ngoại lệ gọn gàng: x == null.
