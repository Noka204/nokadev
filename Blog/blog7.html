Bubbling, capture, delegation: vì sao click bắn “2 lần”?

JS1 DOM Pitfall 25.10.2025 • Đọc nhanh ~ 10 phút

1) Mục tiêu & tóm tắt

Mục tiêu: Hiểu luồng sự kiện (capture/bubble), nhận diện các tình huống click chạy 2 lần, và biết fix mà không “đập” hết event.

Tóm tắt: Trình duyệt đi từ document → target (capture), chạm target, rồi ngược về document (bubble). “Click 2 lần” thường do đăng ký trùng (ở target và parent) hoặc vừa delegation vừa gắn trực tiếp. Dùng once, stopPropagation, hoặc thiết kế selector đúng.

2) Ba pha sự kiện: capture → target → bubble

Pha Mô tả Đăng ký thế nào
Capture Sự kiện đi từ ngoài vào trong el.addEventListener('click', h, { capture: true })
Target Chạm phần tử được click Handler trên chính target
Bubble Đi từ trong ra ngoài el.addEventListener('click', h) (mặc định)

Log thứ tự pha

document.addEventListener('click', e => console.log('doc bubble'))
document.addEventListener('click', e => console.log('doc capture'), { capture: true })
btn.addEventListener('click', e => console.log('btn bubble'))
btn.addEventListener('click', e => console.log('btn capture'), { capture: true })
// Click btn → doc capture → btn capture → btn bubble → doc bubble

Thuật ngữ nhỏ nhưng hay nhầm

  • e.target: phần tử gốc nhận click
  • e.currentTarget: phần tử đang chạy handler

3) Vì sao “click 2 lần” thường gặp?

3.1 Gắn trực tiếp + delegation cùng lúc

// container delegation
list.addEventListener('click', e => {
  const item = e.target.closest('.item')
  if (!item) return
  console.log('container handle item', item.dataset.id)
})

// nhưng lại gắn thêm vào từng .item
items.forEach(it =>
  it.addEventListener('click', e => console.log('item handle', it.dataset.id))
)
// Click 1 item → cả 2 handler đều chạy ⇒ “2 lần”

Fix: chọn 1 trong 2: hoặc delegation hoặc gắn trực tiếp. Nếu buộc phải giữ cả hai, hãy e.stopPropagation() ở handler con.

3.2 Đăng ký handler trùng nhiều lần

function mount() {
  btn.addEventListener('click', onClick) // gọi mount 2 lần → 2 listener
}
const onClick = e => console.log('clicked')
// Fix: dọn trước khi gắn, hoặc dùng { once: true }
btn.removeEventListener('click', onClick)
btn.addEventListener('click', onClick, { once: true })

3.3 Capture + bubble cùng log

btn.addEventListener('click', e => console.log('btn cap'), { capture: true })
btn.addEventListener('click', e => console.log('btn bub'))
// tưởng là “2 lần”, thực ra là 2 pha khác nhau

3.4 Label/checkbox & default action

// click label sẽ kích hoạt click vào input liên kết
label.addEventListener('click', e => console.log('label'))
input.addEventListener('click', e => console.log('input'))
// Thấy 2 log vì default action của label kích hoạt input

Fix: nếu chỉ muốn xử lý ở label, dùng e.preventDefault() hoặc đừng trỏ label vào input.

3.5 Mobile: touch → synthetic click

Nhiều môi trường có thể phát sinh “click ảo” sau touchend. Dùng pointer events hoặc thư viện xử lý nếu gặp case này.

4) Delegation đúng cách

Delegation = gắn 1 listener trên container, lọc bằng selector để xử lý item con. Tiết kiệm bộ nhớ, hoạt động cả với item tạo động. 

list.addEventListener('click', e => {
  const btn = e.target.closest('[data-action]')
  if (!btn || !list.contains(btn)) return

  const { action, id } = btn.dataset
  if (action === 'del') removeRow(id)
  else if (action === 'edit') openEditor(id)
})

Quy tắc vàng: Tránh gắn thêm handler trực tiếp lên item khi đã dùng delegation cho cùng hành vi.

5) Kiểm soát lan truyền

stopPropagation

child.addEventListener('click', e => {
  e.stopPropagation() // chặn bubble lên cha
  doSomething()
})

stopImmediatePropagation

btn.addEventListener('click', e => console.log('A'))
btn.addEventListener('click', e => {
  e.stopImmediatePropagation() // chặn cả listener A/B khác trên cùng phần tử
  console.log('B')
})
btn.addEventListener('click', e => console.log('C')) // không chạy

once: tự tháo listener sau 1 lần

btn.addEventListener('click', e => submit(), { once: true })

6) Default action & passive

preventDefault

link.addEventListener('click', e => {
  e.preventDefault() // chặn điều hướng
  openModal(link.href)
})

Passive listener

// passive true → hứa không gọi preventDefault
window.addEventListener('scroll', e => {
  // làm nhẹ UI, trình duyệt tối ưu cuộn mượt
}, { passive: true })

7) Bài tập tự kiểm

Câu 1 — Tại sao log 2 dòng?

btn.addEventListener('click', e => console.log('cap'), { capture: true })
btn.addEventListener('click', e => console.log('bub'))

Đáp: Hai pha khác nhau (capture & bubble). Không phải “bị gọi hai lần”, mà là hai handler riêng.

Câu 2 — Fix trùng handler

function init() {
  btn.addEventListener('click', onClick)
}
init()
init() // bị gọi lại

Yêu cầu: Đảm bảo chỉ gọi 1 lần dù init chạy lại. Gợi ý: dùng once hoặc removeEventListener trước khi gắn.

// Viết 1 listener trên #list xử lý các nút có data-action="like|share"

8) Kết luận 3 câu

  1. Hiểu 3 pha sự kiện giúp giải thích hầu hết “click chạy 2 lần”.
  2. Đừng vừa delegation vừa gắn trực tiếp cho cùng hành vi — nếu buộc phải có, hãy chặn lan truyền.
  3. Dùng đúng công cụ: stopPropagation, stopImmediatePropagation, once, preventDefault, và passive khi phù hợp.
