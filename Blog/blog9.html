Import default vs named & lỗi đường dẫn “./ vs /”

JS2 Modules Pitfall 25.10.2025 • Đọc nhanh ~ 11 phút

1) Mục tiêu & tóm tắt

Mục tiêu: Phân biệt rõ default vs named, import cho chuẩn, xử lý đúng đường dẫn tương đối/tuyệt đối trên trình duyệt, hiểu khi nào cần import map/bundler, và tránh vòng phụ thuộc khi gom module.

Tóm tắt: Một file có 1 default nhưng nhiều named. Trình duyệt yêu cầu đường dẫn URL hợp lệ (có ./, ../ và đuôi file). Dùng import map hoặc bundler cho “tên gói”. Cẩn thận “barrel” gây vòng tham chiếu.

2) Khái niệm: default vs named export

Default export

// math.js
export default function add(a, b) { return a + b }

// hoặc
function add(a, b) { return a + b }
export default add

Chỉ 1 default trong mỗi file.

Named export

// calc.js
export const PI = 3.14
export function mul(a, b) { return a * b }
// hoặc export { mul, PI }

Có thể nhiều named trong 1 file.

3) Cú pháp import đúng & đổi tên

Import default

import add from './math.js'
console.log(add(1, 2))

Import named

import { mul, PI } from './calc.js'
console.log(mul(2, 3), PI)

Đổi tên khi import

import { mul as multiply } from './calc.js'
multiply(2, 5)

Import cả default lẫn named

import add, { mul } from './bundle.js'

Lỗi thường gặp

  • import add from './calc.js' nhưng calc.js chỉ có named → lỗi vì không có default.
  • import { add } nhưng file export default → tên trong ngoặc xoăn không tồn tại.
  • Gọi import thiếu ./ trên trình duyệt → URL không hợp lệ.

4) Đường dẫn: ./ vs ../ vs / và đuôi .js

Trình duyệt cần URL thật

// ĐÚNG trên trình duyệt
import './setup.js'
import utils from './lib/utils.js'
import api from '../api/index.js'

// SAI: thiếu dấu ./ và đuôi .js (nếu không có server rewrite)
import utils from 'lib/utils'   // bare specifier → cần import map/bundler

Ghi nhớ: Trên trình duyệt, đường dẫn là URL tương đối/tuyệt đối. Thường ./ hoặc ../ và có đuôi file.

“/” là tuyệt đối từ gốc host

// Nếu file đang ở /pages/blog9.html
import thing from '/scripts/thing.js' // từ gốc domain: https://site.com/scripts/thing.js

./ là từ thư mục hiện tại của file; ../ là đi lên 1 cấp.

Đuôi file & index

  • Trên trình duyệt, nhiều server không tự thêm .js. Ghi rõ: ./mod.js.
  • ./dir/index.js ≠ ./dir nếu server không cấu hình “directory index”. Hãy import đường dẫn đầy đủ.

5) Bare specifier, import map & bundler

Bare specifier: import react from 'react'. Trình duyệt thô không hiểu tên gói. Có 2 cách:

Import map (trình duyệt)

<script type="importmap">{
  "imports": {
    "lodash": "/vendor/lodash-es/lodash.js"
  }
}</script>

<script type="module">
  import _ from 'lodash'
</script>

Dùng khi phục vụ file ESM thẳng từ server.

Bundler

Vite/Webpack/Rollup sẽ chuyển bare specifier thành đường dẫn thực, gộp file, tối ưu kích thước, hỗ trợ HMR.

6) Barrel file, vòng phụ thuộc & cách né

Barrel file = file tập trung re-export:

// lib/index.js
export { default as add } from './math.js'
export { mul, PI } from './calc.js'

Import từ barrel

import { add, mul } from './lib/index.js'

Bẫy vòng phụ thuộc

// Nếu math.js lại import từ lib/index.js → vòng tham chiếu
// Triệu chứng: giá trị undefined/partial khi module còn đang khởi tạo

Né: Module “lá” chỉ export; module “barrel” chỉ re-export; đừng để “lá” import ngược “barrel”.

Re-export đổi tên & gộp

// helpers/index.js
export { default as httpGet } from './http.js'
export { format as fmtDate } from './date.js'
export * from './math.js' // gộp named export

7) Bài tập tự kiểm

Câu 1 — Default vs named

// Cho file a.js chỉ có: export default function a(){}
// Viết 2 dòng import: 1 đúng, 1 sai

Câu 2 — Đường dẫn

// File /pages/dashboard/main.js muốn import /scripts/chart.js
// Viết 2 cách import hợp lệ từ main.js

Câu 3 — Barrel an toàn

// Thiết kế bộ lib/ với 3 file: add.js, sub.js, index.js
// Yêu cầu: index.js re-export, tránh vòng phụ thuộc

8) Kết luận 3 câu

  1. Default ≠ named: import đúng cú pháp, có thể import đồng thời nhưng đừng nhầm ngoặc.
  2. Trình duyệt cần URL thật: dùng ./, ../, ghi rõ .js; dấu / là từ gốc host.
  3. Gói = bare specifier: dùng import map hoặc bundler; cẩn thận barrel tạo vòng phụ thuộc.
