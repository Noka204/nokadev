this biến mất: bind/call/apply & khi nào không dùng arrow

JS2 Pitfall 25.10.2025 • Đọc nhanh ~ 10 phút

1) Mục tiêu & tóm tắt

Mục tiêu: Hiểu vì sao this “biến mất”, cách buộc nó quay lại bằng bind/call/apply, và biết lúc nào không nên dùng arrow.

Tóm tắt: this phụ thuộc ngữ cảnh gọi, không phải nơi khai báo. Arrow không có this riêng (lexical), nên rất hợp cho callback, nhưng không hợp cho method cần this, handler DOM cần this, hay constructor/class method prototype.

2) Ý tưởng cốt lõi về this

  • Không phải biến; là tham chiếu ngữ cảnh được gán khi gọi hàm.
  • Arrow lấy this của scope bên ngoài lúc định nghĩa (lexical). Không thể đổi bằng call/apply/bind.

3) 4 quy tắc bind this

Quy tắc Ví dụ this là gì?
Default fn() Trong strict mode: undefined; ngoài strict: window
Implicit obj.fn() obj
Explicit fn.call(obj), fn.apply(obj), fn.bind(obj) Chính obj
Constructor new Fn() Đối tượng mới tạo

4) bind/call/apply khác gì? 

function greet(msg) {
  console.log(msg + ", " + this.name)
}
const bob = { name: "Bob" }

greet.call(bob, "Hi")      // Hi, Bob
greet.apply(bob, ["Hello"])// Hello, Bob

const hiBob = greet.bind(bob, "Xin chào")
hiBob()                    // Xin chào, Bob
  • call: gọi ngay, truyền tham số rời.
  • apply: gọi ngay, truyền mảng.
  • bind: không gọi ngay, trả về hàm mới đã “khóa” this (và có thể khóa sẵn vài đối số).

5) Arrow function & khi nào KHÔNG dùng

5.1 Arrow giữ this theo lexical

const counter = {
  n: 0,
  inc() {
    setTimeout(() => { this.n++ }, 0) // arrow lấy this của inc()
  }
}
counter.inc()
// this.n tăng như mong muốn

5.2 KHÔNG dùng arrow trong các trường hợp sau

  • Định nghĩa method cần this riêng (đặt trực tiếp trong object/prototype).
  • Handler DOM nếu bạn dựa vào this trỏ tới element.
  • Constructor hoặc method cần super.

Sai (arrow làm mất this của method)

const user = {
  name: "Khanh",
  say: () => console.log("Tôi là " + this.name) // this không phải user
}
user.say() // "Tôi là undefined"

Đúng (method thường, arrow cho callback bên trong)

const user = {
  name: "Khanh",
  say() {
    console.log("Tôi là " + this.name)
    ;[1, 2, 3].forEach(v => console.log(v, this.name))
  }
}
user.say()

5.3 DOM: nếu cần this là element, đừng dùng arrow cho handler 

const btn = document.querySelector("#btn")
// Sai: this bên trong arrow không phải btn
btn.addEventListener("click", () => {
  console.log(this) // window/undefined (strict)
})

// Đúng: function thường lấy this = element
btn.addEventListener("click", function () {
  this.classList.add("active")
})

6) Thực chiến & refactor

6.1 “this biến mất” khi tách method ra biến

const obj = {
  n: 1,
  get() { return this.n }
}
const g = obj.get
// g() gọi theo Default binding → this = undefined
// g() // ReferenceError (nếu truy cập this.n)

Gỡ bằng bind

const gSafe = obj.get.bind(obj)
gSafe() // 1

6.2 Mượn method giữa các object

function total() {
  return this.items.reduce((s, x) => s + x, 0)
}
const cart = { items: [5, 15, 20] }
total.call(cart) // 40

6.3 Khoá ngữ cảnh cho callback API

class Timer {
  constructor() {
    this.ticks = 0
    this.tick = this.tick.bind(this) // khoá this
  }
  start() {
    this.id = setInterval(this.tick, 1000)
  }
  tick() {
    this.ticks++
  }
}

6.4 Arrow vs bind cho callback

class Timer2 {
  ticks = 0
  start() {
    this.id = setInterval(() => { this.ticks++ }, 1000) // arrow lấy this của instance
  }
}

7) Bài tập tự kiểm

Câu 1

const u = {
  name: "Lan",
  say() { console.log(this.name) }
}
const s = u.say
s()

Hỏi: In ra gì và vì sao? Sửa trong một dòng.

Đáp án: Không in “Lan” vì default binding. Sửa: const s = u.say.bind(u).

Câu 2

const list = {
  items: ["a", "b"],
  show() {
    setTimeout(function () {
      console.log(this.items) // undefined
    }, 0)
  }
}
list.show()

Sửa bằng 1 trong 2 cách:

// Cách 1: bind
setTimeout(function () { console.log(this.items) }.bind(list), 0)

// Cách 2: arrow
setTimeout(() => { console.log(this.items) }, 0)

Câu 3

const btn = document.querySelector("#save")
btn.addEventListener("click", () => {
  this.classList.add("busy")
})

Vì sao sai? Arrow không có this element. Sửa: 

btn.addEventListener("click", function () {
  this.classList.add("busy")
})

8) Kết luận 3 câu

  1. this phụ thuộc cách gọi, không phải nơi khai báo.
  2. bind/call/apply buộc this về đúng đối tượng; bind trả hàm mới, call/apply gọi ngay.
  3. Arrow không có this riêng: tuyệt vời cho callback lồng trong method, nhưng tránh dùng làm method, handler DOM phụ thuộc this, hay constructor.
